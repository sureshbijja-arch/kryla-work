Hardened

Kryla Design System & Template UX Overhaul

The system elevates average content. A member with three photos looks like they hired an agency. Restraint is the richness — not decoration.

What survived

Direction

Build a system so well designed that average content looks intentional. Confident negative space and typography do the work — not gradients, glows, or animated orbs.

Design Modes — new design_mode column

craft (baker, salon, chef, trainer) — warm spacing, 1.333× type ratio, approachable.
editorial (photographer, doctor, musician) — generous whitespace, 1.414× ratio, photography-forward.
Set at onboarding from persona. Palette stays color-only. These are separate concerns.

Token Implementation

CSS custom properties as source of truth — [data-mode='craft'] { --type-display: 3.5rem; --space-section: 5rem; }. Tailwind config extends the same scale. data-mode on the LayoutRenderer wrapper. Both layers — not either.

auto Variant — render time, not DB

LayoutRenderer reads member content and picks the hero variant:
gallery photo exists → photo (full-bleed)
avatar only → split or centered
no photos → dark (typographic)
Admin can override. auto is the default for all new section entries.

Retrofit before production data

All section components rewritten on the token system before any member has sections JSON in the DB. No migration debt, no live pages depending on current ad-hoc values.

Build sequence

W1
Foundation — invisible but non-negotiable

CSS custom properties, design_mode column, persona→mode mapping at onboarding, data-mode on LayoutRenderer wrapper. Nothing looks different yet.

W2
Hero rewrite + auto variant — first visible win

HeroSection on tokens. auto variant logic in LayoutRenderer. Bakers with gallery photos get full-bleed photo hero automatically. Most members look noticeably better without any admin action.

W3
All remaining sections on tokens

Services, Highlights, Bio, Gallery, FAQ, Contact. Consistent spatial rhythm. Pages feel composed, not assembled.

W4
Persona default section arrays — product promise lands

Default sections JSON for each persona. New member's page looks custom-made on day one, auto variant throughout, no admin panel needed.

What was killed

✕

Decoration as richness — animated orbs, gradient borders, glow pulses. 2019 agency portfolio moves. Killed.

✕

Encoding design_mode in palette — palette is color only. Spatial grammar is a separate concern. Killed.

✕

Extending template to carry design mode — too coupled. New column. Killed.

✕

Temporary bridge — deriving design_mode from template at render time. Killed. Proper column from day one.

✕

Writing auto variant back to DB — render time resolution only. DB stays clean. Killed.
