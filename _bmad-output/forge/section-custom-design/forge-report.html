Section Custom Design

Forged — June 2026  ·  Pressure-tested to hardened

What Survived

Core Principle

Design freedom constrained so bad output is structurally impossible. Member pride and business quality are both required — they stop conflicting when the constraints are right.

UI Home

Section styling lives inline in the Page Layout tab, expanding on each section card alongside the existing variant picker. No new tab.

Background Colour

16 curated presets (Warm Cream, Cool Linen, Slate Blue, Forest Mist, Rose Blush, Amber Glow, Charcoal, Midnight, Ocean, Sage, Terracotta, Lavender, Sand, Steel, Ivory, Graphite). System auto-detects luminance → sets text dark or white. Member never touches contrast.

Background Photo

Member uploads any photo on hero · services · highlights · bio. Auto-scrim overlay guarantees text readability. Not available on faq or contact.

Moving Frames (Hero only)

Toggle on/off, pick 1 / 2 / 3 frames. Populated from gallery images (Grow+ dependency — frames are Grow+ only).

Plan Ladder

TierSection styling access
SeedControls visible, greyed out + upgrade prompt
Sprout+16 background colour presets + photo upload (hero / services / highlights / bio)
Grow+Everything above + moving frames on hero (1–3, from gallery)

Killed and Why

  • Video backgrounds — LCP killer, CDN cost, bad-output risk too high for any tier
  • Per-section text editing — already covered by the AI chat editor
  • Custom hex / colour picker — unconstrained input is a bad-output factory
  • Per-section font override — breaks design system coherence across sections
  • Per-section accent colour override — produces incoherent pages
  • Background photo on FAQ / contact — kills text readability and form usability
  • Moving frames on non-hero sections — no clear value, bad-output risk

Implementation Shape

Data model

Extend pages.sections jsonb: { sectionKey, variant, order, style?: { bg?: { type: 'color'|'photo', value: string }, frames?: { enabled: boolean, count: 1|2|3 } } }

Save flow

Style changes write to draft_data → preview → publish. Same flow as AI chat edits. Not direct-save.

New upload type

type=section-bg added to /api/my-space/upload

Seed gate

opacity-50 pointer-events-none + upgrade nudge — same pattern as the existing Layouts tab gate
